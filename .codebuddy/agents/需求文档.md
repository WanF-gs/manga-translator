仿网易云音乐平台需求分析文档
一、项目概述
1.1 项目名称
仿网易云音乐 Web 平台（核心功能版）
1.2 项目背景
本项目为技术学习型项目，目标是复刻网易云音乐官网的核心页面与基础能力，落地前后端分离的全栈开发方案，验证 Vue + FastAPI + MySQL 技术栈的工程化实现效果。
1.3 建设目标
前端 1:1 还原网易云音乐「发现」首页视觉效果与基础交互
实现完整的用户账号体系：邮箱验证码注册、用户名 / 邮箱密码登录
实现底部全局音乐播放控件，对接网易云 API 完成音乐播放能力
构建高性能、可扩展的前后端分离架构，代码结构规范可维护
1.4 需求范围
已包含范围
前端页面：网易云音乐「发现（discover）」首页 UI 复刻、底部全局音乐播放控件、用户登录模块、用户注册模块
后端服务：用户认证接口、邮箱验证码服务、首页音乐数据代理接口、歌曲播放地址代理接口
数据存储：用户账号信息存储、邮箱验证码记录存储
第三方对接：对接网易云音乐公开 API，实现音乐数据查询与音频播放代理
不包含范围
歌单详情页、专辑详情页、歌手主页、全量排行榜、搜索结果页等所有跳转子页面
评论、收藏、下载、自建歌单、个性化推荐等用户个性化功能
音乐文件存储、转码、分发服务，所有音频资源均来自第三方 API
后台管理系统、运营配置系统、数据统计功能
1.5 参考依据
设计与开发以网易云音乐官网发现页 https://music.163.com/#/discover 为唯一视觉参考，页面内区块、布局、配色、组件样式严格对齐参考页。
二、功能性需求
2.1 前端页面模块
2.1.1 首页（发现页）UI 复刻
页面整体采用固定宽度居中布局，核心区块与参考页完全对齐，包含以下模块：
顶部导航栏：固定在页面顶部，包含平台 logo、主导航菜单（仅 UI 展示，无页面跳转）、搜索框、用户登录 / 头像入口
热门推荐区：区块标题 +「更多」按钮，横向排列歌单卡片，展示歌单封面与歌单名称
新碟上架区：区块标题 +「更多」按钮，网格布局展示新专辑卡片，展示专辑封面、专辑名称、歌手名称
榜单区：区块标题 +「更多」按钮，并列展示飙升榜、新歌榜、原创榜三个子榜单，每个榜单展示 Top10 歌曲条目，包含排名、歌曲名、歌手名
所有非首页跳转入口仅做 UI 占位，不实现页面跳转逻辑
2.1.2 底部音乐播放控件
全局固定在页面底部，常驻显示，页面滚动 / 刷新不中断播放，核心功能：
歌曲信息展示：歌曲封面图、歌曲名称、歌手名称
播放控制：上一首、播放 / 暂停状态切换、下一首
进度控制：播放进度条实时更新，支持拖动进度条跳转播放位置，展示当前播放时长与歌曲总时长
音量控制：音量调节滑块、静音切换按钮
交互触发：点击首页榜单内的歌曲，可切换当前播放曲目并自动播放
2.1.3 用户登录页面 / 弹窗
参考网易云登录交互，支持弹窗或独立路由两种形式，核心功能：
登录方式：支持「用户名 + 密码」「邮箱 + 密码」两种登录方式，共用一个输入框自动识别
表单校验：非空校验、邮箱格式合法性校验
错误反馈：账号不存在、密码错误、账号禁用等场景的明确文案提示
登录态同步：登录成功后关闭弹窗，顶部导航栏更新为用户头像与用户名
跳转注册：提供「去注册」入口，切换到注册模块
2.1.4 用户注册页面 / 弹窗
核心功能：
注册方式：邮箱验证码注册
表单字段：邮箱地址、验证码、登录密码、确认密码
表单校验：邮箱格式校验、密码强度校验、两次密码一致性校验、验证码有效性校验
验证码发送：点击「获取验证码」按钮向邮箱发送 6 位数字验证码，按钮进入 60 秒倒计时，限制发送频率
注册成功后自动跳转至登录环节，或直接完成登录并返回首页
2.2 用户认证系统模块
2.2.1 账号登录功能
支持用户名或邮箱作为账号标识，后端统一校验
密码校验采用哈希比对，不接触明文密码
登录成功后签发 JWT 令牌，返回用户基础信息
支持登录态过期自动失效
2.2.2 邮箱注册功能
校验邮箱唯一性，同一邮箱不可重复注册
校验验证码有效性与时效性，验证通过后方可完成注册
密码自动加密后存入数据库，不存储明文
注册成功自动生成用户名，支持后续修改
2.2.3 邮箱验证码服务
生成 6 位纯数字随机验证码
验证码有效期 5 分钟，过期自动失效
发送频率限制：同一邮箱 1 分钟内最多发送 1 次，1 小时内最多发送 5 次
验证码单次有效，使用后立即标记为已使用，不可重复校验
2.2.4 身份鉴权机制
采用 JWT 无状态鉴权方案，令牌有效期 7 天
需要登录态的接口校验请求头中的令牌，无效或过期返回统一错误码
前端请求自动携带令牌，过期后自动跳转登录
2.3 音乐服务模块
2.3.1 首页数据获取
热门推荐歌单数据：对接第三方 API 获取官方推荐歌单列表
新碟上架数据：获取最新发行专辑列表
榜单数据：分别获取飙升榜、新歌榜、原创榜的 Top10 歌曲数据
后端统一做数据格式清洗与转换，对齐前端 UI 渲染所需字段
2.3.2 音乐播放能力
根据歌曲 ID 获取可播放的音频 URL
支持单曲播放、暂停、进度跳转、音量调节
播放列表支持上下首切换，初始播放列表为当前榜单歌曲
2.3.3 第三方 API 对接
后端统一代理所有网易云 API 请求，解决前端跨域问题
对第三方接口做异常兜底，请求失败返回友好提示
接口字段映射适配，第三方接口变更时仅需后端调整，不影响前端逻辑
三、非功能性需求
3.1 UI 还原度要求
页面布局、配色体系、字体字号、组件间距、交互样式与网易云官网发现页视觉一致性达到 95% 以上，无明显视觉差异。
3.2 性能需求
首页首屏加载时间 ≤ 2 秒（网络正常环境下）
后端基础接口平均响应时间 ≤ 200ms（第三方 API 转发耗时除外）
歌曲切换播放无明显卡顿，音频加载时间 ≤ 1 秒
3.3 兼容性需求
兼容 Chrome、Edge、Firefox 等主流浏览器最新两个大版本
页面适配 1366px、1920px 等主流桌面分辨率，最小支持宽度 1200px
暂不做移动端适配
3.4 安全性需求
用户密码采用 bcrypt 不可逆哈希加密存储
后端所有接口做参数合法性校验，防范 SQL 注入、XSS 攻击
验证码接口增加频率限制，防范恶意刷取
JWT 令牌密钥安全存储，不可硬编码在代码中
跨域配置白名单限制，禁止非法域名请求接口
3.5 可维护性需求
前后端代码分层清晰，遵循单一职责原则
接口、字段、组件命名规范统一，关键逻辑添加注释
配置项与业务代码分离，支持环境切换
四、技术架构设计
4.1 整体架构
采用前后端分离架构：
前端 Vue 应用负责页面渲染、交互逻辑、状态管理，通过 HTTP 请求调用后端接口
后端 FastAPI 服务提供 RESTful 接口，处理业务逻辑、操作数据库、代理转发第三方音乐 API
MySQL 8.0 持久化存储用户与验证码数据
音频资源直接从网易云 CDN 加载，不经过后端转发

六、后端接口设计
6.1 接口规范
统一接口前缀：/api/v1
请求数据格式：application/json
响应数据格式：统一 JSON 结构 { "code": 状态码, "message": 提示信息, "data": 业务数据 }
鉴权方式：需要登录的接口请求头携带 Authorization: Bearer {token}
6.2 用户认证接口
1. 发送注册验证码
请求方式：POST
接口路径：/api/v1/auth/send-register-code
请求参数：
表格
参数名	类型	必填	说明
email	string	是	目标邮箱地址
响应数据：发送成功提示
2. 用户注册
请求方式：POST
接口路径：/api/v1/auth/register
请求参数：
表格
参数名	类型	必填	说明
email	string	是	注册邮箱
code	string	是	邮箱验证码
password	string	是	登录密码
confirm_password	string	是	确认密码
响应数据：注册成功提示
3. 用户登录
请求方式：POST
接口路径：/api/v1/auth/login
请求参数：
表格
参数名	类型	必填	说明
account	string	是	用户名或邮箱
password	string	是	登录密码
响应数据：
表格
参数名	类型	说明
access_token	string	JWT 访问令牌
token_type	string	令牌类型，固定为 Bearer
user_info	object	用户基础信息（id、用户名、邮箱、头像）
4. 获取当前用户信息
请求方式：GET
接口路径：/api/v1/auth/me
鉴权要求：需要登录
响应数据：当前登录用户的基础信息
6.3 音乐数据接口
1. 获取热门推荐歌单
请求方式：GET
接口路径：/api/v1/music/recommend-playlists
请求参数：limit（可选，默认 10）
响应数据：歌单列表，包含歌单 ID、封面 URL、歌单名称、播放量
2. 获取新碟上架列表
请求方式：GET
接口路径：/api/v1/music/new-albums
请求参数：limit（可选，默认 10）
响应数据：专辑列表，包含专辑 ID、封面 URL、专辑名、歌手名
3. 获取榜单详情
请求方式：GET
接口路径：/api/v1/music/top-list
请求参数：type（必填，榜单类型标识：飙升榜 / 新歌榜 / 原创榜对应 ID）
响应数据：榜单名称、歌曲列表（排名、歌曲 ID、歌曲名、歌手名、时长）
4. 获取歌曲播放地址
请求方式：GET
接口路径：/api/v1/music/song-url
请求参数：id（必填，歌曲 ID）
响应数据：歌曲音频播放 URL
5. 获取歌曲详情
请求方式：GET
接口路径：/api/v1/music/song-detail
请求参数：ids（必填，歌曲 ID，多个用逗号分隔）
响应数据：歌曲详情（名称、歌手、封面、时长）
七、前端详细设计
7.1 路由设计
表格
路由路径	页面名称	说明
/	首页重定向	自动跳转至发现页
/discover	发现首页	核心页面，包含所有首页区块与播放控件
其他路径	404 重定向	统一跳转至发现页
登录、注册采用弹窗形式，不占用独立路由，减少页面跳转，对齐网易云交互体验。
7.2 首页布局结构
采用上中下三层固定布局：
顶部导航栏：高度 60px，固定定位，层级 100，页面滚动始终可见
主体内容区：宽度 1200px 水平居中，顶部预留 60px 内边距，按「热门推荐 → 新碟上架 → 榜单」垂直排列
底部播放栏：高度 53px，固定定位在页面底部，层级 100，全局常驻，主体内容区底部预留 53px 内边距避免遮挡
7.3 状态管理设计
基于 Pinia 管理两个全局 Store：
userStore：存储登录态、token、用户基础信息，提供登录、登出、获取用户信息方法
playerStore：存储当前播放歌曲、播放列表、播放状态、播放进度、音量，提供播放、暂停、切歌、进度跳转、音量调节方法

九、风险说明与约束
第三方 API 风险：本项目对接的网易云音乐 API 为非官方公开接口，存在接口变更、失效、访问频率限制等风险，仅可用于技术学习，不可用于生产环境。
版权约束：项目所有音乐资源版权归网易云音乐及对应版权方所有，仅用于学习演示，禁止商业用途与非法传播。
UI 还原约束：以当前参考页面版本为准，若网易云官网后续更新页面样式，还原度以当前需求确认的版本为准。
邮件服务风险：公共邮箱 SMTP 服务存在发送限额与垃圾邮件拦截可能，正式使用需接入第三方邮件服务商。
