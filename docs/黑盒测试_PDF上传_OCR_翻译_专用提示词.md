## 一、测试纪律（铁则）

1. **浏览器操作 ONLY**：所有测试交互必须通过 Playwright MCP 在真实浏览器中完成。不允许写 Python/JS 测试脚本、不允许用 curl/httpie 发请求、不允许直接调 API 接口。唯一允许的后台操作是步骤 0 的服务状态检查。
2. **黑盒原则**：不读任何业务源码（`.py`、`.tsx`、`.go`），仅以浏览器页面显示和交互结果为准。
3. **素材隔离**：100% 使用 `测试项目/` 目录内 PDF 文件。只读不修改原始素材。
4. **每步必截图**：每一次表格中的测试用例执行后立即调用 `playwright_screenshot` 截图留存。截图保存到本轮测试报告目录下的 `screen/` 文件夹，文件名格式：`模块编号-步骤描述-时间戳.png`（如 `01-单页PDF上传成功-20260630-1430.png`）。
5. **Bug 三要素**：严重等级（P0/P1/P2/P3）+ 复现步骤 + 实际现象 vs 预期效果 + 截图凭证。
6. **空壳拦截**：页面元素存在 ≠ 功能可用。交互流程走不到终点的一律标 P1。
7. **PDF格式全覆盖**：单页PDF、多页PDF（5页+）、扫描版PDF 三种必须各自独立测试独立截图。

## 六、异常情况处理

### 6.1 后端未响应
若某项操作（OCR/翻译）点击后长时间无响应：
1. 打开 `playwright_console_logs` 查看浏览器控制台是否有报错
2. 在终端执行 `wsl curl -s http://localhost:8080/health` 确认网关状态
3. 若后端确实挂了，重启全栈：`wsl bash /mnt/c/Users/WanFi/Desktop/大三实训/demo_04/start_wsl2_all.sh`
4. 标记为 **【后端阻塞-待联调】**，与前端 Bug 分开记录

### 6.2 页面崩溃/白屏
立即 `playwright_screenshot` 截图记录，然后 `playwright_navigate` 重新加载页面。如反复出现，标 P0。

### 6.3 功能按钮不见
先用 `playwright_get_visible_html` 确认 DOM 中是否有该元素。如果元素存在但不可见（隐藏/CSS 遮盖），标 P1 空壳功能。
