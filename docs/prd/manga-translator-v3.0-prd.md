## 一、需求描述

### 1.1 背景

#### 业务问题
全球漫画（日漫、韩漫、国漫等）爱好者面临语言障碍，非官方翻译版本质量参差不齐、更新滞后。个人爱好者缺乏专业翻译/嵌字技能，现有机器翻译工具效果粗糙，无法满足高质量阅读与收藏需求。**2026年，多模态LLM技术成熟，为漫画翻译带来革命性突破，但竞品仍停留在纯文本翻译阶段，缺乏端到端的一站式解决方案。**

#### 目标用户

| 用户群体 | 核心特征 | 核心诉求 | 新增关注点 |
|----------|----------|----------|----------------|
| 个人动漫爱好者 | 无专业翻译/嵌字技能，希望低门槛看懂生肉漫画 | 上传即翻译，成品质量对标汉化组 | AI语气匹配、一键风格保持 |
| 外语学习者 | 有双语对照学习需求 | 双语对照阅读、单词即点即译 | 社交化学习、语法解析、艾宾浩斯复习 |
| 业余内容创作者 | 追求成品质量与阅读体验 | 高清导出、多格式支持、样式自定义 | 批量校对工作台、AI排版引擎 |
| **漫画出版社（B端）** | 有正规授权内容的批量翻译需求 | 高效率、术语统一、质量可控 | API接入、团队协作、专属术语库 |
| **汉化组/翻译团队** | 多人协作翻译，需要进度管理 | 协作校对、版本管理、批量处理 | 轻量协作、评论标注、质量报告 |

#### 价值主张
打造「上传即翻译、一键出成品、多端无缝用、**AI持续进化**」的轻量化网页端工具，依托云端算力实现专业级翻译与画质效果。从个人工具进化为创作者生态平台。

### 1.3 用户场景

#### 场景1：爱好者快速看懂生肉漫画
> 小王是日漫爱好者，下载了最新一话的日文生肉。他在电脑上打开系统，上传CBZ压缩包，选择「一键翻译」模式，系统自动完成检测→翻译→回填。他在预览界面快速浏览确认后，导出中文版CBZ，系统同时生成翻译质量报告，全程不到5分钟。

#### 场景2：外语学习者双语对照阅读
> 小张在学习日语，她在移动端上传了日文漫画，选择双语对照模式。翻译完成后，她在阅读器中点击任意气泡可切换原文/译文，点击单词弹出释义，将生词加入生词本。她还可以使用艾宾浩斯复习模式记忆单词，查看语法解析辅助学习。打开有声剧场模式，听角色用不同声音朗读对话。

#### 场景3：创作者精细化嵌字
> 小陈是业余汉化组成员，他在电脑端使用「专业编辑」模式，手动调整文字选区、校对翻译结果、微调字体样式和排版。新版校对工作台让他可以双栏对照快速校对200页漫画，键盘Tab键快速切换区域，批量替换统一术语，效率提升300%。

#### 场景4：移动端碎片化阅读
> 小赵在通勤路上用手机打开系统，浏览之前电脑端处理好的漫画作品。得益于PWA 2.0，他在地铁无网络环境下仍可离线阅读已缓存的作品，竖屏卷轴滚动阅读，夜间模式下舒适看完一整话。

#### 场景5：汉化组协作翻译
> 「极速汉化组」有5名成员，正在协作翻译一部200话的长篇漫画。主编创建作品并上传原始文件，分配章节给不同成员。成员A检测文字区域，成员B负责翻译校对，成员C负责嵌字排版。页面级锁定避免编辑冲突，评论标注功能让成员可以针对特定对话讨论翻译方案，变更日志记录每次修改。

#### 场景6：出版社批量授权翻译
> 某国内漫画出版社获得了一部日本漫画的授权，需要翻译成简体中文。出版社使用API平台将200话内容批量接入翻译流水线，统一术语库确保人名/地名/招式名翻译一致。翻译完成后自动生成多语言版本的电子漫画书，同时导出翻译质量报告供编辑审核。

#### 场景7：社交媒体传播 [NEW]
> 小刘翻译完一话热血漫画后，使用动态漫画模式生成了一个30秒的短视频——镜头自动推进到打斗场景，对话气泡依次弹出，打击感画面微微震动，拟声词带脉冲特效。她导出视频分享到小红书，获得大量点赞，朋友通过视频中的二维码扫码进入平台。

### 2.1 图像导入与个人项目管理模块

#### 2.1.1 多格式批量导入 [P0]

**功能描述**：支持多种格式的漫画文件导入，自动解析并生成项目结构。

**用户故事**：
> 作为漫画爱好者，我希望能够上传各种格式的漫画文件（图片、压缩包、PDF），以便系统能自动识别并组织为可处理的页面。

**验收标准**：
- [ ] 电脑端支持单张/批量 JPG、PNG、WebP 图片上传（单次最多200张）
- [ ] 电脑端支持 CBZ、ZIP、RAR、7Z 压缩包上传，自动解压并按文件名排序生成页面
- [ ] 电脑端支持 PDF 文件导入，自动逐页拆分为独立页面
- [ ] 电脑端支持拖拽上传和本地文件夹选择
- [ ] 移动端支持相册批量选图（最多50张）
- [ ] 移动端支持拍照直接上传
- [ ] 移动端支持压缩包文件导入
- [ ] 所有导入方式在上传过程中显示进度条
- [ ] 导入完成后自动生成缩略图列表

**异常处理**：
- 不支持的格式：弹窗提示「不支持的文件格式：xxx，支持的格式为 JPG/PNG/WebP/CBZ/ZIP/RAR/7Z/PDF」
- 压缩包损坏：提示「文件已损坏，无法解压，请重新上传」
- 单张图片超过50MB：提示「图片过大（>50MB），建议压缩后上传」
- 上传中断：已上传部分保留，支持断点续传

**验收标准**：

### 2.2 文字区域检测与OCR识别模块

#### 2.2.1 全类型文字区域检测 [P0]

**功能描述**：自动检测漫画中的各类文字区域并精准定位。

**验收标准**：
- [ ] 自动检测对话气泡、内心独白、旁白框、拟声词、效果字五类文字区域
- [ ] 精准区分文字内容与气泡边框边界
- [ ] 检测结果以矩形/多边形选区标注在图像上
- [ ] 支持选区颜色编码区分文字类型
- [ ] 检测置信度低于60%的区域高亮提醒用户复核

**业务规则**：
- 气泡边框与文字区域的边距保留≥2px，确保后续擦除不伤及框线

> **实现增强（优于PRD）**：文字检测已集成真实 **Comic Text Detector（YOLO+DBNet 混合，漫画专训）ONNX 模型**（`ctd_detector.py`），非通用 OpenCV 阈值检测；模型缺失时诚实返回空而非伪造框。检测输出对气泡类文字用 `cv2.findContours`/`convexHull` 生成真实多边形轮廓（`boundary_mode=polygon`），联动 §2.2.8 的选区最小化。

---

#### 2.2.2 特殊排版适配 [P0]

**功能描述**：适配漫画中各类非标准文字排版。

**验收标准**：
- [ ] 支持日文竖排文本检测与识别
- [ ] 支持弧形文本检测（沿气泡弧度排列的文字）
- [ ] 支持倾斜文本检测（倾斜角度±45°以内）
- [ ] 支持艺术变形文本检测（拟声词特效字）
- [ ] 不规则气泡形状（星形、云形等）内的文字区域正确框选

---

#### 2.2.3 区域智能过滤 [P0]

**功能描述**：自动跳过无需翻译的区域。

**验收标准**：
- [ ] 默认跳过页码、刊头、作者签名区域
- [ ] 支持用户自定义过滤规则（按位置、按大小、按类型）
- [ ] 过滤规则可保存为预设

---

#### 2.2.4 选区手动调整 [P0]

**功能描述**：支持手动编辑自动检测的文字选区。

**验收标准**：
- [ ] 电脑端：支持矩形选区拖拽调整大小/位置
- [ ] 电脑端：支持多边形选区节点编辑
- [ ] 电脑端：支持选区合并（选中多个选区→合并为一个）
- [ ] 电脑端：支持选区拆分（将一个选区拆分为多个）
- [ ] 电脑端：支持微调内边距（独立调整上下左右）
- [ ] 电脑端：支持批量添加/删除文字区域
- [ ] 移动端：支持基础选区拖拽调整

#### 2.2.8 检测选区视觉影响最小化策略 [P0]

**功能描述**：确保 OCR 文字区域检测框和擦除回填 mask 在功能准确的前提下，最小化对漫画画面的视觉遮挡和修改范围。漫画翻译工具的核心差异化体验点——不同于通用 OCR 工具可用简单矩形框，本系统面对的是视觉艺术作品，检测框/mask 的视觉侵入性直接决定成品质量和用户观感。

**设计原则**：
1. **气泡贴合原则**：有气泡边框的文字，检测选区应紧贴气泡内轮廓（非简单矩形），不遮挡气泡外画作
2. **最小包围原则**：无气泡的文字（拟声词/效果字/旁白），检测选区应为文字轮廓的紧密包围多边形，最小化覆盖面积
3. **Mask 精准原则**：擦除和回填的 mask 应精确到文字像素级，不过度修改背景
4. **全链路一致原则**：检测 → 擦除 → 回填三个阶段共享同一精密轮廓，避免三重叠加破坏

**业务规则**：
- 有气泡的文字选区：使用多边形/贝塞尔曲线贴合气泡内轮廓，与气泡边框保持 ≥2px 安全边距
- 无气泡的文字选区：使用文字轮廓的凸包/紧密包围多边形，覆盖面积 ≤ 文字实际面积的 120%
- 选区展示半透明（默认透明度 30%），不遮挡下方画面细节
- 编辑模式下支持一键隐藏/显示所有选区线（快捷键 H）
- 擦除 mask 精度达文字轮廓 ±2px，多余修改面积 ≤ 文字面积的 10%
- 检测阶段即区分"有气泡文字"与"无气泡文字"，采用不同包围策略
- 检测框输出支持矩形（降级方案）和多边形/贝塞尔曲线（优选方案）两种模式

**验收标准**：
- [ ] 有气泡场景：选区覆盖面积 ≤ 气泡内面积的 95%（不超出气泡边界，紧贴内轮廓）
- [ ] 无气泡场景：选区覆盖面积 ≤ 文字实际面积的 120%（紧密包围，矩形仅为降级方案）
- [ ] 选区展示：默认半透明（透明度30%），不遮挡下方画面细节
- [ ] 编辑模式下快捷键 H 一键隐藏/显示所有选区线
- [ ] 擦除 mask：多余修改面积 ≤ 文字面积的 10%，不修改文字外的背景像素
- [ ] 检测框输出同时支持多边形/贝塞尔曲线（优选）和矩形（降级），API 返回中标注包围模式类型
- [ ] 所有视觉元素相关的生成/修改操作均遵守"最小改动面积"原则

**视觉影响度量**：
| 场景类型 | 选区覆盖约束 | 擦除 mask 约束 | 展示形式 |
|----------|--------------|----------------|----------|
| 有气泡文字（圆形/椭圆/矩形气泡） | 选区覆盖 ≤ 气泡内面积 95% | 多余修改 ≤ 文字面积 10% | 贴合气泡内轮廓的多边形 |
| 无气泡文字（旁白/内心独白） | 选区覆盖 ≤ 文字实际面积 120% | 多余修改 ≤ 文字面积 10% | 紧包围多边形 |
| 拟声词/效果字（艺术字） | 紧密包围多边形 | 边缘精度 ±2px | 文字轮廓凸包 |

> **跨模块影响说明**：本策略直接影响检测模块（§2.2）、擦除模块（§2.4）、排版引擎（§2.4.3）、校对工作台（§2.11）。各模块需基于同一精密轮廓进行操作，形成"检测一次、全程复用"的全链路闭环。

### 2.4 智能背景修复与文字回填模块

#### 2.4.0 智能擦除引擎增强 [P0] [NEW v3.0]

**功能描述**：在 v1.0 基础擦除上，增加分背景类型的量化质量标准和擦除失败降级策略。擦除是翻译质量的基石——擦不好，后续排版和回填全白搭。与 §2.2.8 的精密 mask 联动，实现从检测到擦除的像素级精度闭环。

> **联动 §2.2.8**：擦除阶段复用检测阶段的精密轮廓 mask，确保擦除范围 = 文字区域 ±2px，不额外修改背景。检测阶段产出的 mask 直接作为擦除的输入，避免二次检测带来的精度损失。

**验收标准**：
- [ ] 网点纸区域擦除后纹理连续，网点密度与原背景一致，无明显糊化或断层
- [ ] 排线/速度线区域擦除后线条连续，方向、粗细与原线条一致，无断层
- [ ] 渐变背景擦除后过渡自然，无色阶断裂，颜色偏差 ΔE ≤ 3
- [ ] 纯色背景擦除后颜色均匀，与周围区域色差 ΔE ≤ 2
- [ ] 擦除边缘与周围背景融合，无明显边界线或光晕
- [ ] 擦除失败自动标记：擦除区域与原背景 SSIM ≤ 0.85 时自动标记为"需人工修复"
- [ ] 人工擦除引导：标记区域支持一键进入手动修补模式，高亮显示问题区域
- [ ] 擦除质量评分：每页自动生成擦除质量分（0-100），低于60分提示用户复核

**分背景类型擦除质量标准**：
| 背景类型 | 擦除后要求 | 失败判定标准 | 检测方法 |
|----------|-----------|-------------|----------|
| 网点纸（スクリーントーン） | 纹理连续，网点密度一致 | SSIM < 0.80 | SSIM + 人眼抽查 |
| 排线/速度线 | 线条连续，方向一致 | 线条断裂长度 > 5px | Canny边缘连续性检测 |
| 渐变背景 | 过渡自然，无色阶断裂 | 色阶断裂 ≥ 3处 | 梯度直方图分析 |
| 纯色背景 | 颜色均匀，无明显色差 | ΔE > 3 | CIEDE2000色差公式 |
| 复杂纹理（云/水面等） | 纹理自然，无重复模式 | 肉眼可见修复痕迹 | LPIPS感知相似度 |


### 2.28 动态漫画模式 [P2~P3] [NEW v3.0]

**功能描述**：不是做动画，而是给静态漫画注入"微动感"，创造介于漫画和动画之间的全新阅读体验。同时支持导出动态漫画短视频，实现社交传播裂变。

**核心体验**：

**智能分镜平移（Ken Burns 效果）**：
- [ ] AI 自动识别画面焦点（人物面部/动作中心/对话框），生成镜头缓慢推进/拉远的动画轨迹
- [ ] 支持3种运镜模式：缓慢推进(默认)、焦点跟随、分格依次展示
- [ ] 运镜速度和焦点可手动调整

**气泡逐条弹出**：
- [ ] 对话气泡按阅读顺序依次淡入+微放大弹出，模拟"边读边展开"的节奏感
- [ ] 气泡弹出动画时间可调（默认每个气泡间隔0.5秒）
- [ ] 支持调整气泡弹出顺序（按阅读顺序、按对话逻辑、自定义）

**关键帧微动**：
- [ ] 对画面中可分离的关键元素（如飘落的樱花、流动的云、火焰）做2-3帧循环微动
- [ ] 基于AI前景/背景分离技术，对前景元素做轻微位移或缩放循环
- [ ] 微动幅度可控（5档：极其微妙/轻微/适中/明显/强烈）

**震动/冲击特效**：
- [ ] 打击/爆炸场景自动添加画面微震效果（3档强度）
- [ ] 拟声词带脉冲缩放效果（与 §2.14.1 拟声词风格保持联动）
- [ ] 场景转换添加过渡动画（淡入淡出/划像/百叶窗等）

**导出功能**：
- [ ] 导出为 MP4 短视频（15-60秒可调，支持自定义分辨率）
- [ ] 导出时自动添加水印和"由XX翻译"的品牌标识
- [ ] 一键分享到社交平台（微信/微博/QQ/抖音/小红书）
- [ ] 分享卡生成：带封面+二维码的推广图片

**商业化潜力**：这是一个极强的差异化功能。"静态转动态"在社交媒体上的传播性极强，可做成"导出动态漫短视频→分享→引流回平台"的病毒式增长闭环。免费版每次导出带水印，专业版无水印+更高分辨率。

> **实现说明（P2~P3 已提前落地核心）**：MP4 导出已是**真实 ffmpeg 视频合成**（`export_service/api/audio_dynamic.py`）——图片序列拼接 + Ken Burns `zoompan` 运镜 + libx264 编码 + 可选 AAC 语音轨，Redis 任务状态追踪、MinIO 上传、前端真实轮询下载。尚未实现的是气泡逐条弹出动画、关键帧微动、震动特效、水印与分享卡。

## 五、接口需求

### 5.1 接口设计原则

- RESTful API 风格
- JSON 数据格式
- HTTPS 加密传输
- JWT Token 认证
- 统一错误响应格式

### 5.2 统一响应格式

```json
{
  "code": 0,
  "message": "success",
  "data": {},
  "request_id": "uuid"
}
```

**错误码规范**：
| 错误码 | 说明 |
|--------|------|
| 0 | 成功 |
| 1001 | 参数校验失败 |
| 1002 | 资源不存在 |
| 1003 | 权限不足 |
| 1004 | 免费版功能限制 |
| 1005 | 文件格式不支持 |
| 1006 | 文件大小超限 |
| 1007 | 页面已被其他用户锁定 [v2.0] |
| 1008 | API配额已用完 [v2.0] |
| 1009 | 团队人数已达上限 [v2.0] |
| 1010 | 离线操作排队中 [v2.0] |
| 2001 | 认证失败 |
| 2002 | Token过期 |
| 3001 | 处理任务失败 |
| 3002 | 处理超时 |
| 3003 | GPU资源紧张，已加入优先队列 [v2.0] |
| 3004 | 端侧模型加载失败，已切换云端处理 [v2.0] |

### 6.4 内容安全与合规 [ENHANCED v3.0]

**违规内容检测**：
- 上传时自动检测明显违规内容（色情、暴力、恐怖主义），拦截并提示用户
- 违规内容检测采用业界标准模型，误拦截率≤5%
- 被拦截的内容仅做检测标记，不存储原文

> **实现说明**：图像/文本审核已接入**腾讯云内容安全 IMS/TMS**（TC3-HMAC-SHA256 签名，`common/clients/content_safety.py`），通过 Celery 异步触发（`moderation_tasks.py`）。⚠️ 当前策略为 fail-open（凭证缺失或调用异常时放行），尚未在上传主链路做同步强制拦截，UGC 敏感词本地过滤与 PIPL 数据导出/账号注销仍待补全。

**UGC内容审核**：
- 公开词典的术语贡献需经过自动审核（敏感词过滤）+ 人工抽查
- 用户自定义角色名称/口头禅需过滤违规内容
- 声音克隆功能需用户确认合规使用条款

**版权合规**：
- 不做版权审核（内容归属用户自行负责）
- 用户上传字体需自行确认版权，系统标注版权信息提示
- 声音克隆功能提示用户遵守声音权相关法规

**数据隐私合规**：
- 遵循中国《个人信息保护法》（PIPL）要求
- 用户可随时导出或删除全部个人数据
- 隐私政策需明确说明数据使用范围
- 不将用户翻译内容用于公开训练（除非用户明确同意加入改进计划）

## 七、设计决策与约束

### 7.1 架构决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 部署形态 | B/S网页端 + PWA 2.0离线能力 | 免安装、跨平台、增强移动端体验 |
| 算力架构 | 云端统一调度 + 端云协同推理(P2) | 重计算任务不适端侧，端侧加速轻量检测 |
| 翻译引擎 | 内置双引擎 + API开放 + 模型路由 | 降低门槛、构建生态、成本优化 |
| 存储策略 | 云端归档+7天临时清理 + 本地离线缓存(PWA) | 平衡便利性与成本，离线可用 |
| AI模型迭代 | 主动学习闭环 | 持续提升翻译质量 |
| 商业化 | Freemium + 订阅混合 | 提升LTV和收入持续性 |
| 团队功能 | 轻量协作 | 覆盖汉化组和B端场景 |
| 认证方式 | JWT Token | 无状态、适合前后端分离、跨端友好 |
| 任务处理 | 异步队列 | 批量任务后台运行，不阻塞用户操作 |

### 7.2 约束条件

- 必须兼容 Chrome/Edge/Safari/Firefox 最新两个正式版本
- 移动端必须兼容 iOS 14+ 和 Android 10+
- 所有用户上传内容默认仅本人可见
- 不支持离线AI处理核心链路（需联网使用云端算力），但支持离线阅读
- 免费版单批次最多50页，专业版200页
