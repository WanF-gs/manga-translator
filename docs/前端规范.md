### 2.2 目录结构原则

1. **按功能模块组织**，而非按文件类型组织
2. **共享组件** 放在 `components/` 下，按业务领域分子目录
3. **页面专用组件** 与页面文件放在一起，使用 `_components/` 目录
4. **每个组件一个目录**，包含 `index.tsx`、`style.module.css`、`types.ts`
5. **API服务层** 按后端微服务模块划分

### 3.2 变量命名

| 类型 | 规范 | 示例 |
|------|------|------|
| 变量 | camelCase | `currentPage`, `userName` |

## 四、代码风格规范

### 4.1 TypeScript

- 所有新文件必须使用 `.tsx` / `.ts`
- 禁止使用 `any`，使用 `unknown` + 类型守卫
- 接口命名使用 `I` 前缀或直接 PascalCase（团队约定：不强制I前缀）
- 类型导出使用 `export interface` / `export type`
- 优先使用 `type` 定义联合类型、交叉类型
- 优先使用 `interface` 定义对象结构

## 五、组件开发规范

### 5.1 组件设计原则

1. **单一职责**：一个组件只做一件事
2. **可复用性**：通过 props 控制变化，避免硬编码
3. **可组合性**：小组件组合成大组件
4. **可测试性**：纯展示组件与容器组件分离

### 5.2 组件分类

| 分类 | 说明 | 示例 |
|------|------|------|
| **基础UI组件** | 通用UI元素，无业务逻辑 | Button, Modal, Toast |
| **通用业务组件** | 跨页面复用的业务组件 | FileUpload, StatusBadge |
| **页面组件** | 特定页面的根组件 | ProjectListPage, EditorPage |
| **布局组件** | 页面布局框架 | PcLayout, MobileLayout |

### 5.3 组件文件结构

```
ComponentName/
```

### 5.4 Ant Design 使用规范

- 统一从 `antd` 导入，避免路径导入

- 表单使用 `Form.useForm()` + `onFinish` 模式

- 移动端使用 `antd-mobile` 对应组件

## 七、接口请求规范

### 7.1 axios实例配置

```typescript
// services/request.ts
import axios, { AxiosError, InternalAxiosRequestConfig } from 'axios';
import { message } from 'antd';
import { authStore } from '@/stores/authStore';

const request = axios.create({
  baseURL: process.env.NEXT_PUBLIC_API_BASE_URL,
  timeout: 30000,
  headers: {
    'Content-Type': 'application/json',
  },
});

// 请求拦截器
request.interceptors.request.use(
  (config: InternalAxiosRequestConfig) => {
    const token = authStore.getState().accessToken;
    if (token) {
      config.headers.Authorization = `Bearer ${token}`;
    }
    // 移动端标识
    if (typeof window !== 'undefined' && window.innerWidth < 768) {
      config.headers['X-Device-Type'] = 'mobile';
    }
    return config;
  },
  (error) => Promise.reject(error)
);

// 响应拦截器
request.interceptors.response.use(
  (response) => {
    const { code, message: msg } = response.data;
    if (code !== 0) {
      // 业务错误处理
      handleBusinessError(code, msg);
      return Promise.reject(new Error(msg));
    }
    return response.data;
  },
  (error: AxiosError) => {
    // 网络/HTTP错误处理
    handleHttpError(error);
    return Promise.reject(error);
  }
);
```

### 7.3 统一错误处理

```typescript
// utils/errorHandler.ts
import { message } from 'antd';
import { authStore } from '@/stores/authStore';

const ERROR_MESSAGES: Record<number, string> = {
  1001: '参数校验失败，请检查输入',
  1002: '请求的资源不存在',
  1003: '权限不足',
  1004: '该功能需要升级高级版',
  1005: '不支持的文件格式',
  1006: '文件大小超出限制',
  2001: '用户名或密码错误',
  2002: '登录已过期，请重新登录',
  5000: '服务器繁忙，请稍后重试',
};

export function handleBusinessError(code: number, msg: string) {
  if (code === 2002) {
    authStore.getState().logout();
    return;
  }
  message.error(ERROR_MESSAGES[code] || msg || '未知错误');
}

export function handleHttpError(error: AxiosError) {
  if (!error.response) {
    message.error('网络连接失败，请检查网络');
    return;
  }
  // 根据HTTP状态码处理
  switch (error.response.status) {
    case 401:
      authStore.getState().logout();
      break;
    case 403:
      message.error('无权访问');
      break;
    case 500:
      message.error('服务器错误');
      break;
    default:
      message.error('请求失败');
  }
}
```

## 八、状态管理规范

### 8.1 Zustand Store规范

```typescript
// stores/authStore.ts
import { create } from 'zustand';
import { persist } from 'zustand/middleware';

interface AuthState {
  accessToken: string | null;
  refreshToken: string | null;
  user: UserData | null;
  isAuthenticated: boolean;
  // Actions
  login: (token: string, refreshToken: string, user: UserData) => void;
  logout: () => void;
  setUser: (user: UserData) => void;
}

export const useAuthStore = create<AuthState>()(
  persist(
    (set) => ({
      accessToken: null,
      refreshToken: null,
      user: null,
      isAuthenticated: false,

      login: (accessToken, refreshToken, user) =>
        set({ accessToken, refreshToken, user, isAuthenticated: true }),

      logout: () =>
        set({ accessToken: null, refreshToken: null, user: null, isAuthenticated: false }),

      setUser: (user) => set({ user }),
    }),
    {
      name: 'auth-storage',
      partialize: (state) => ({
        accessToken: state.accessToken,
        refreshToken: state.refreshToken,
        user: state.user,
        isAuthenticated: state.isAuthenticated,
      }),
    }
  )
);
```

### 8.2 状态管理原则

1. **服务端状态**：使用 `@tanstack/react-query` 管理（缓存、同步、重试）
2. **客户端状态**：使用 `Zustand` 管理（UI状态、编辑器状态）
3. **URL状态**：使用 Next.js `useSearchParams` / `useParams`
4. **表单状态**：使用 Ant Design `Form.useForm()`

```typescript
  return useMutation({
    mutationFn: projectApi.create,
    onSuccess: () => {
      queryClient.invalidateQueries({ queryKey: ['projects'] });
    },
  });
}
```

## 十、安全规范

### 10.1 XSS防护

- React JSX 默认转义，不直接使用 `dangerouslySetInnerHTML`
- 用户输入的翻译文本需做 HTML 实体编码
- 文件上传校验文件类型白名单

### 10.2 Token安全

- Access Token 存储在内存中（Zustand persist 到 localStorage 仅用于刷新）
- Refresh Token 存储在 httpOnly Cookie 中
- Token 过期前自动刷新

### 10.3 敏感信息

- 不在前端代码中硬编码 API Key、密钥
- 使用环境变量 `NEXT_PUBLIC_*` 仅存放非敏感配置
- 用户个人信息脱敏展示

### 11.1 组件优化

- 使用 `React.memo` 避免不必要的重渲染

- 大列表使用虚拟滚动（react-window / antd Table 虚拟滚动）

## 十二、测试规范

### 12.1 测试层级

| 层级 | 工具 | 覆盖率目标 |
|------|------|-----------|
