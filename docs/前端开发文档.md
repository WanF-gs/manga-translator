## 二、编辑器页面架构

### 2.1 页面结构

```
┌──────────────────────────────────────────────────────┐
│ Toolbar（顶部工具栏：返回/模式/撤销重做/保存/导出）  │
├──────────┬──────────────────────────────┬────────────┤
│ Sidebar  │       Canvas 画布区          │ Property   │
│ 章节列表 │  ┌──────────────────────┐   │ Panel      │
│ + 页面   │  │  漫画页面图片        │   │ 属性面板   │
│ 缩略图   │  │  + RegionOverlay     │   │            │
│          │  │    (选区矩形+标签)    │   │            │
│          │  └──────────────────────┘   │            │
│ 240px    │        flex-1               │  320px     │
├──────────┴──────────────────────────────┴────────────┤
│ StatusBar（底部状态栏：页码/状态/进度/缩放/尺寸）     │
└──────────────────────────────────────────────────────┘
```

### 2.2 组件职责

| 组件 | Props | 功能 |
|------|-------|------|
| `Canvas` | imageUrl, imageWidth, imageHeight, regions, selectedRegionId, showOriginal, scale, onScaleChange, onSelectRegion | 画布渲染：图片展示 + 选区覆盖层 + Ctrl滚轮缩放 + 空格拖拽平移 |
| `RegionOverlay` | regions, selectedRegionId, showOriginal, scalePercent, onSelect | 选区矩形绘制：类型标签、锁定指示器、选中高亮 |
| `Sidebar` | chapters, currentPageId, onSelectPage, onTogglePanel | 章节分组可折叠 + 页面缩略图列表 + 状态图标 |
| `Toolbar` | projectName, currentPageNumber, totalPages, onToggleShowOriginal, onAutoTranslate, onSave, onExport | 顶部工具栏：模式切换(Ant Segmented)、撤销重做、导出下拉 |
| `PropertyPanel` | region, onUpdate, onDelete, onToggleLock, onApplyAll, onClose | 右侧属性面板：字体/字号/颜色/对齐/锁定/删除 |
| `StatusBar` | currentPageNumber, totalPages, pageStatus, regionCount, progress, scale, onScaleChange, imageWidth, imageHeight, activeStep, isProcessing | 底部状态栏：页码/状态/处理流程/缩放滑块/尺寸 |

### 2.3 数据流

```
URL params (projectId)
  → projectApi.getDetail() + getChapters() 获取项目+章节
  → pageApi.getList(chapterId) 获取页面列表
  → pageApi.getDetail(pageId) 获取页面详情+文字区域
  → useEditorStore 管理编辑器状态
  → 组件通过 props 接收数据，通过回调更新状态
```

## 三、认证与路由守卫

### 3.1 Middleware 逻辑

```
请求进入 → 检查路径是否公开（/login, /register, /api, /_next）
  ├── 公开路径 → 放行
  ├── 已登录 + 访问登录页 → 重定向到 /pc 或 /m
  ├── 未登录 + 受保护页面 → 重定向到 /login（附带 redirect 参数）
  └── 根路径 / → 根据 UA 重定向到 /pc 或 /m
```

### 3.2 Token 刷新机制

```
请求 → 响应拦截器
  ├── code=0 → 正常返回
  ├── code=2002 或 HTTP 401 → 触发刷新
  │   ├── 无刷新进行中 → 调用 authApi.refreshToken()
  │   │   ├── 成功 → 更新 store token → 重放失败请求
  │   │   └── 失败 → 清除状态 → 跳转登录页
  │   └── 已有刷新进行中 → 加入等待队列 → 刷新完成后重放
  └── 其他错误 → 显示错误信息
```

## 四、状态管理

### 4.1 authStore（Zustand + persist）

```typescript
interface AuthState {
  accessToken: string | null;
  refreshToken: string | null;
  user: UserData | null;
  isAuthenticated: boolean;
  // 持久化到 localStorage key: "manga-auth"
}
```

### 4.2 editorStore（Zustand）

```typescript
interface EditorState {
  currentProjectId / currentPageId;  // 编辑上下文
  canvasScale / canvasPosition;      // 画布状态
  selectedRegionId / regions;        // 选区状态
  mode: 'simple' | 'professional';   // 编辑模式
  showOriginal;                      // 显示模式
  history / historyIndex;            // 操作历史（最多20条）
  undo() / redo();                   // 撤销/重做
  // 细粒度更新：updateRegion / updateRegionBoundary / updateRegionStyle
}
```

## 五、API 服务层

### 5.1 请求封装特性

- 自动添加 `Authorization: Bearer token` 请求头
- 自动添加 `X-Device-Type`（pc/mobile）
- 自动添加 `X-Request-ID` 追踪ID
- 统一错误处理（网络/HTTP/业务错误）
- Token 过期自动刷新（并发安全）
- 响应数据自动解包（code=0 时返回 data）

### 5.2 API 模块

| 模块 | 主要方法 |
|------|----------|
| `authApi` | login, register, refreshToken, logout, getProfile |
| `projectApi` | getList, getDetail, create, update, delete, toggleFavorite, getChapters, createChapter, updateChapter, deleteChapter, sortChapters |
| `pageApi` | getList, getDetail, upload, delete, sort, updateStatus, detectRegions, updateRegions, runOCR, translate, inpaint, render, batchProcess, retry |

## 六、开发指南

### 6.1 启动开发服务器

```bash
cd manga-translator-web
npm install
npm run dev
# 访问 http://localhost:3000
```

### 6.2 Mock API

项目内置 Next.js Route Handler Mock API（`src/app/api/v1/[...path]/route.ts`），覆盖所有接口，无需后端即可开发。

后端就绪后，删除该文件或设置 `NEXT_PUBLIC_API_BASE_URL` 环境变量指向真实后端。

### 6.3 添加新页面

1. 在 `src/app/pc/` 或 `src/app/m/` 下创建新目录和 `page.tsx`
2. 使用 `'use client'` 指令（如需客户端交互）
3. 使用 `projectApi`/`pageApi` 等已封装 API
4. 使用 Tailwind CSS 类名 + Ant Design 组件
5. 处理 loading/empty/error 三种状态

### 6.4 编辑器组件使用示例

```tsx
import { Canvas, Sidebar, Toolbar, PropertyPanel, StatusBar } from '@/components/editor';

// 三栏布局
<div className="h-screen flex flex-col">
  <Toolbar projectName="..." ... />
  <div className="flex-1 flex">
    <Sidebar chapters={...} currentPageId={...} ... />
    <Canvas imageUrl="..." regions={...} ... />
    <PropertyPanel region={...} ... />
  </div>
  <StatusBar ... />
</div>
```
